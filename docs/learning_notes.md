# 学习笔记

记录学习数学建模与科学计算过程中的感悟和经验。

## 入门阶段：从Python到NumPy

最开始学Python的时候，觉得它就是一个"高级计算器"。直到接触到NumPy，才发现向量化运算的威力——一个矩阵乘法就能替代好几层for循环。

Pandas让数据处理变得很直观，Matplotlib让结果可视化变得简单。这三个库是所有后续实验的基础。

## 数值计算：理解"近似"的意义

学数值积分的时候有个很深的感触：数学分析里那些"精确解"在实际问题中往往是不存在的。数值方法的核心思想就是"用有限逼近无限"——矩形公式、梯形公式、Simpson公式，本质上都是对积分的离散化近似。

非线性方程求解也是类似。Newton迭代法看起来简单，但收敛性依赖于初值选择，这个在实际使用中经常遇到。

## 优化建模：把问题"翻译"成数学语言

线性规划是让我第一次感受到"建模"魅力的方法。一个看起来很复杂的实际问题（比如生产计划、运输调度），只要能把它"翻译"成目标函数+约束条件的标准形式，就能用工具直接求解。

难点不在于调用linprog函数，而在于：
1. 如何定义决策变量
2. 如何构造目标函数
3. 如何把各种限制条件写成约束

这个"翻译"能力是数学建模的核心。

## 微分方程：从解析到数值

SymPy的dsolve能求很多微分方程的符号解，这很爽。但实际建模中遇到的微分方程大多没有解析解，必须用数值方法。

Euler法是最直观的——把微分变成差分，一步步往前走。但精度太差。改进的Euler法（预测-校正）和Runge-Kutta方法在精度和计算量之间做了很好的平衡。

Lorenz混沌系统的实验让我印象特别深：两个初值只差0.0001，但轨迹随时间急剧分离。这就是"蝴蝶效应"的数学本质——确定性系统中的内在随机性。

## 图论：算法之美

图论部分是最有"算法感"的。Dijkstra算法的手写实现让我真正理解了贪心策略的精妙——每次选离起点最近的未访问节点，逐步扩展最短路径。

Floyd-Warshall算法的三重循环初看很暴力，但"经过k个中间节点"的递推思想很优雅。

最有意思的是把实际问题建模为图论问题：设备更新问题→最短路，选址问题→重心法，PageRank→有向图的稳态分布。

## 机器学习：数学原理比调参重要

学SVM的时候，如果只是调用sklearn.svm.SVC，几行代码就完了。但理解最大间隔、核函数、对偶问题的数学推导，才是真正有价值的部分。

PCA也是。理解了协方差矩阵的特征值分解，才能明白"降维"的数学本质是保留数据方差最大的方向。

## 一些经验

1. **先手写再调库**：Dijkstra、Floyd这些经典算法，先手写一遍再用NetworkX，理解深度完全不一样
2. **注释很重要**：隔一个月再看自己写的代码，没有注释根本看不懂
3. **可视化是最好的debug**：画个图比print一堆数字直观得多
4. **数学推导不能省**：代码只是工具，背后的数学原理才是核心
5. **多做对比实验**：比如比较Euler法和odeint的精度差异，比只看理论更有收获
